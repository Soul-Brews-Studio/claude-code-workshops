# Two-Issue Pattern

ใช้ 2 issues แยกกัน: Context + Plan

## Why?
- แยก context dump ออกจาก actionable tasks
- Clean task tracking
- Better organization

## Pattern
```
Issue #1: context: Session state 2025-12-23
Issue #2: plan: Implement feature X
```
