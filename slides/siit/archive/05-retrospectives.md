---

### SLIDE 2: Why Retrospectives
```
Title: Why rrr Matters
- Capture what actually happened
- Turn mistakes into rules
- Improve CLAUDE.md over time
- Build team memory
```

---

### SLIDE 3: When to Write
```
Title: The Right Moment
- End of every session
- After a major task
- After a failure or surprise
Short, honest, consistent
```

---

### SLIDE 4: Recommended Structure
```
Title: Retro Template
1) Summary
2) Wins
3) Issues
4) Lessons learned
5) Next actions
Keep it under 10 minutes
```

---

### SLIDE 5: Example
```
Title: Example Retro (Short)
- Summary: Added auth endpoint
- Win: clear API design
- Issue: missing tests
- Lesson: add test checklist
- Next: update CLAUDE.md
```

---

### SLIDE 6: Distillation Loop
```
Title: Retro -> Rules
- Extract lessons
- Convert to CLAUDE.md rules
- Update short codes if needed
Result: workflow improves itself
```

---

### SLIDE 7: Pattern Detection
```
Title: Look for Patterns
- Repeated mistakes
- Slow steps
- Missing docs
- Tooling gaps
Patterns = high-value fixes
```

---

### SLIDE 8: Anti-Patterns
```
Title: Avoid These
- Blame-only retros
- Vague notes
- No actions
- Never updating CLAUDE.md
```
